# Tech Stack

## Language & Runtime
- Go 1.26.1 (module: `github.com/skip-the-line`)

## Key Libraries
| Library | Purpose |
|---|---|
| `go-chi/chi/v5` | HTTP router |
| `google/go-github/v62` | GitHub webhook parsing and API client |
| `slack-go/slack` | Slack API client (DMs, user lookup) |
| `caarlos0/env/v11` | Environment variable config parsing |
| `go.uber.org/zap` | Structured logging |
| `go.opentelemetry.io/otel` | Metrics and tracing (OTLP/gRPC export) |
| `gopkg.in/yaml.v3` | Subscription config parsing |
| `matryer/moq` | Interface mock generation |

## Common Commands

```bash
make build        # Build binary (CGO_ENABLED=0)
make test         # Run all unit tests
make test-cover   # Run tests + generate HTML coverage report
make lint         # Run golangci-lint
make generate     # Regenerate moq mocks
make up           # Start full stack via Docker Compose
make down         # Stop Docker Compose stack
make logs         # Tail app logs
make send-event EVENT=pull_request  # Send a sample webhook to local app
```

## Build
- Binary name: `skip-the-line` (output path configurable, default `./skip-the-line`)
- CGO disabled: `CGO_ENABLED=0 go build -o skip-the-line ./cmd/server`
- Docker: multi-stage build, final image uses `gcr.io/distroless/static:nonroot`

## Local Development
- Docker Compose spins up the app + two `mendhak/http-https-echo` mock servers (Slack on `:3000`, GitHub on `:4000`)
- App is reachable at `http://localhost:8080`
- Set `SLACK_API_URL` and `GITHUB_API_URL` env vars to point at mock servers instead of real APIs

## Testing
- Standard `go test ./...` — no external test framework
- Table-driven tests with `t.Run` subtests
- Mocks generated by `moq` into `internal/mocks/`, imported as `github.com/skip-the-line/internal/mocks`
- Use `noop` OTel providers in tests (e.g. `noop.NewMeterProvider()`)

## Agent Instructions
- Always use `make build` to build and `make test` to run tests — never raw `go build` or `go test` commands directly
