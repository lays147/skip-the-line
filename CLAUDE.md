## Architecture

**Core flow:** `GitHub webhook â†’ HMAC signature validation â†’ event routing â†’ subscriber resolution â†’ Slack DM`

**Entrypoint** (`cmd/server/main.go`): loads config, constructs logger, wires all dependencies via constructor injection, registers chi routes, handles graceful shutdown. No global state.

**Event pipeline:**
1. `webhook.Handler` validates the HMAC signature, parses the raw payload into a typed `go-github` struct, and calls `NotificationServicer.Notify`
2. `notification.NotificationService` routes by event type/action, resolves recipients (individuals + team members via `GitHubTeamResolver`), then calls `SlackNotifier` for each
3. Subscriber resolution uses `subscription.Registry` (embedded `subscriptions.yaml`) to map GitHub username â†’ email, then `SlackNotifier.LookupUserByEmail` â†’ `SendDM`

**Slack messages** are built as Block Kit JSON using `[]any` maps and marshalled to string.

**Errors from Slack delivery are best-effort:** logged with `zap.Warn`/`zap.Error` and skipped â€” never returned to the caller or the webhook response.

## Key Conventions

**Interfaces live on the consumer side.** `notification/recipients.go` defines `GitHubTeamResolver` and `SlackNotifier`; the `github` and `slack` packages implement them without knowing the interface exists.

**Mocks** are generated by `moq` into `internal/mocks/` with `package mocks`. The `//go:generate` directives live in `recipients.go` and `service.go` next to the interface they mock. Regenerate with `make generate` after any interface change. Never edit mock files manually.

**Logger** (`cmd/server/logger.go`): production JSON zap logger writing to stdout, teed with an `otelzap` bridge. Log level comes from `cfg.LogLevel` (`LOG_LEVEL` env var). `service` and `environment` fields are added as defaults. The OTel core injects `trace_id`/`span_id` when a span is active.

**Config** is loaded once at startup via `config.Load()` and passed down by value â€” never accessed globally.

**`subscriptions.yaml`** is embedded at compile time; adding/removing subscribers requires a rebuild.

**Errors** all errors should be wrapped using %v. Errors inside the same package doesn't need to be wrapped.

## Testing Patterns

- Table-driven tests with `t.Run` subtests throughout
- Black-box test packages (`package notification_test`) for public API; white-box (`package webhook`) when unexported helpers are needed
- Use `noop` OTel providers in tests: `noop.NewMeterProvider()`, `noop.NewTracerProvider()`
- Mocks from `internal/mocks` implement the interfaces via generated `moq` code

## Local Development

Docker Compose starts the app plus two `mendhak/http-https-echo` mock servers (Slack on `:3000`, GitHub on `:4000`). Set `SLACK_API_URL` and `GITHUB_API_URL` to redirect calls to those mocks instead of the real APIs.

Required env vars: `GITHUB_WEBHOOK_SECRET`, `GITHUB_TOKEN`, `SLACK_BOT_TOKEN`. Copy `.env.example` to `.env` to get started.
